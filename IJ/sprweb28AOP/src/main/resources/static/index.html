AOP 연습

AOP -> Aspect - Oriented Programming 관점 지향 프로그래밍이라고 불리는 프로그래밍 패러다임입니다. 핵심 기능과 관련 없는 부가적인 기능을 모듈화 하여 코드 중복을 줄이고 유지보수성을 향상시키는데 사용됩니다. 즉, 공통관심사를 분리하여 핵심 로직과 분리관리하는 것을 목표로 합니다. AOP 핵심개념: 관점(Aspect): 핵심 기능과 독립적인 부가기능 (예: 로깅,보안,트랜잭션). 횡단 괌심사(Cross-cutting Concern): 애플리케이션 여러 곳에서 반복적으로 사용되는 기능. 엮기(Weaving): 관점의 로직을 다른 클래스에 적용하는 과정. 관심사 분리(Separation of Concerns): 핵심 로직과 부가 기능을 분리하여 모듈화하는 과정. AOP 장점: 코드 중복 감소: 부가 기능을 한곳에 모아 관리하여 중복 코드를 줄일 수 있습니다. 유지보수성 향상: 핵심 로직과 부가 기능을 분리하여 변경이 용이합니다. 모듈성 증가: 각 관점을 기준으로 코드를 모듈화하여 코드의 구조를 명확하게 만들 수 있습니다. AOP 예시: 로깅: 애플리케이션의 모든 메서드 호출을 로깅할 떄, AOP를 이용하여 로깅 로직을 한곳에 모아 관리하고. 모든 메서드 호출 전에 로깅 로직을 적용할 수 있습니다. 보안: 사용자 인증 및 권한 검증 로직을 AOP를 사용하여 한 곳에 모아 관리하고, 특정메서드 호출 전에 인증 및 권한 검증 로직을 적용할 수 있습니다. 트랜잭션: 데이터베이스 트랜잭션 처리를 AOP를 사용하여 한 곳에 모아 관리하고, 특정 메서드 호출 전에 트랜잭션을 시작하고, 호출 후 트랜잭션을 커밋하거나 롤백할 수 있습니다. AOP는 주로 스프링 프레임워크에서 많이 사용되며, 스프링 AOP는 횡단 관심사를 모듈화하여 코드의 중복을 줄이고 유지보수성을 향상시키는 데 도움을 줍니다.
